## Purpose

The v1 frontend for eTape: a TypeScript + React + Vite SPA served by the Go engine,
covering monitoring (multi-symbol charts, scanner, movers, news) and trading (focused
symbol charts, L2 ladder, time & sales, account/positions/orders, hotkey-driven order
entry). Replaces Earl's two ZeroPro trading monitors; the third (browsing) monitor is
out of scope.

## Decisions made during brainstorming

| Question | Decision |
|---|---|
| Windowing | Two browser windows (one per monitor), same SPA, `?workspace=` selects layout; no SharedWorker; no native multi-window in v1 (Wails later is a packaging step) |
| Layout flexibility | Dockable panels inside each window (dockview): drag-rearrange, resize, stacked tabs; layouts auto-saved |
| Workspaces | Workspaces are **named documents** `{name, panels[], layout, panelConfigs}` persisted engine-side; "Monitoring" and "Trading" ship as seed documents; minimal Add Panel menu in v1; workspace manager UI later |
| Execution scope | Display surfaces + order ticket + **customizable hotkeys** with sizing modes; ladder click-trading excluded from v1 |
| Symbol linking | ZeroPro-style **link groups** (colored channels); panels assigned to a group follow its focused symbol; unassigned panels are pinned |
| Chart library | **Unforked** Lightweight Charts v5 (≥5.2.0) + custom marker plugin (diamond, border) ported from the `earlisreal-lightweight-charts` fork; no more fork maintenance |
| Indicators | Catalog (VWAP, EMA, SMA, MACD, volume, per-bar buy/sell delta), customizable per chart (add/remove instances, periods, colors); computed engine-side, streamed as series; no scripting engine |
| Time & sales | Canvas painter over a ring buffer (deviation from stack doc's "virtualized list" — consistent with painter architecture, burst-proof; loses native text selection, accepted) |
| Data plane | Per-window WebSocket to engine; plain-TS stores; one rAF scheduler per window; React renders chrome only |
| Connection visibility | Dedicated Connection Status panel: per-link latency (UI↔engine, engine↔moomoo, engine↔TZ) + engine-persisted event log |

Rejected alternatives: SharedWorker connection broker (extra hop, WKWebView risk,
solves a non-cost on loopback); single-window v1 (workflow genuinely spans two
monitors); free-floating OS windows (browser popups are janky; revisit with Wails v3);
single-global-focus and explicit-promotion symbol linking (link groups subsume both).

## Architecture

```
Go engine ── serves ──▶ http://127.0.0.1:PORT   (built static UI + WS endpoint)
                          │
        ┌─────────────────┴──────────────────┐
        ▼                                    ▼
  Window A  ?workspace=monitoring      Window B  ?workspace=trading
  (own WS connection)                  (own WS connection)
        └──────── BroadcastChannel ──────────┘
             link-group focus events only — never market data
```

One SPA in `ui/` (Vite + React + TS; types generated from Go structs via tygo).
Production: engine serves the built files; launching eTape = start engine, open two
browser windows. Four layers per window, strict dependency direction
(`chrome → render → data → wire`, never backwards):

1. **`wire/`** — WS client: connect, reconnect with jittered backoff, snapshot-then-
   delta per topic, decode to tygo types. Subscribes only to topics the window's
   visible panels need. The engine owns all moomoo subscription-quota decisions; the
   UI requests logical topics and never thinks about quota.
2. **`data/`** — plain TypeScript stores, zero React: `BookStore`, `TapeRing` (ring
   buffer), `BarStore` (per symbol+timeframe), `QuoteStore`, `ExecStore`,
   `ScannerStore`, `NewsStore`, `HealthStore`. Updated at message rate; each marks a
   dirty flag.
3. **`render/`** — pure canvas painters (`paint(ctx, state)`): ladder, tape, chart
   marker/overlay plugins; plus one **rAF scheduler** per window that paints every
   dirty surface once per frame and clears flags. All viewport/coordinate math
   (index↔x, price↔y, pan/zoom clamping, nice-number axis) lives in pure unit-tested
   modules — ported from wickplot's `CandlestickChartMath` / `ChartViewport`.
4. **`chrome/`** — React: dockview shell, panel frames, order ticket, settings,
   tables. Exec topics arrive engine-coalesced (account ~4 Hz, positions ~100 ms
   batches, orders event-driven) — low-rate enough for React state.

**Hard rule (restated from stack doc):** high-frequency data never flows through React
state. Chart/ladder/tape are canvas surfaces mounted once via refs, painted
imperatively, coalesced to one repaint per rAF tick.

**Link groups:** each panel header shows a group swatch (or none = pinned). Focus
changes publish `{group, symbol}` on `BroadcastChannel("etape.link")` (cross-window)
and echo to the engine (persistence + pre-subscription). A symbol box in each
workspace header types-to-focus a group.

**Workspace persistence:** dockview layout JSON + panel configs auto-save to the
engine on change (debounced). Panels must survive drag/resize: stable React keys (no
re-mount on move), ResizeObserver per panel driving canvas/chart resize.

## Panels

Every panel: declares its topics, gets a dockview slot, registers painters with the
scheduler if it paints, renders an inline error card if it throws (error boundary per
panel — one broken panel never takes down the ticket).

### Monitoring workspace (seed layout, monitor 1)

- **Chart × 4** — pinned, own symbol + timeframe each. Same chart component as below.
- **Scanner** — pre-market rank table (~35–70 rows after filters, ~2 s engine
  refresh): symbol, % change (3-digit-safe), last price, float (compact `2.1M`;
  "no print yet" state, never fake 0%), pre-market volume. New-hit rows flash;
  already-seen symbols muted (dedup resets at ET midnight). User-configurable
  min-%-change, float cap (nullable = off), volume floor. Session-aware header
  (Waiting / Polling pre-market / RTH mode) + last-refresh time. Row click focuses
  its link group. Same component parameterized by session (pre/after-hours/RTH rank
  APIs share payload shape).
- **Movers list** — the scanner component, RTH parameterization, sorted by % change.
- **News** — headline list for its group's focused symbol: headline, source,
  `seen_at` time (labeled as seen-time, not publish time), click opens URL. Renders
  the broker-agnostic `NewsItem` shape only. Visual slot reserved for high-salience
  halt banners (v2 feed).
- **Connection Status** (seeded here, addable anywhere) — see below.

### Trading workspace (seed layout, monitor 2)

- **Chart × 4** — all in the focus group; seed timeframes 1m / 10s / 5m / 60m (the
  fourth flips 60m↔D as needed); each switchable across 10s, 1m, 5m, 15m, 30m, 60m,
  D, W, M.
- **L2 ladder** — custom canvas: 10 levels per side, price/size/cumulative, last-trade
  flash, working orders marked at their price levels (display only in v1). US symbols
  only (LV3 entitlement); non-US shows a "no depth entitlement" state.
- **Time & sales** — canvas tape over `TapeRing`: time/price/size, colored
  BUY/SELL/NEUTRAL, minimum-size filter, pause-on-scroll (auto-resume on jump-to-live).
- **Account bar** — equity, buying power, day P&L (TZ-authoritative for the gate;
  unrealized marked from moomoo ticks), realized, **armed/disarmed state + arm
  control**, connection dots (summary of Connection Status; click opens the panel).
- **Positions** — rows with live unrealized P&L; flatten button per row (routes
  through the gate like any order).
- **Open orders** — 9-state lifecycle incl. `PendingNew` (in-flight) and `Replacing`;
  per-row cancel; cancel-all; reject reasons verbatim (R-codes); `StreamGap`
  reconcile badge.
- **Order ticket** — see Order entry.

### Connection Status panel

- **Latency rows per link:** UI↔engine (app-level WS ping every ~2 s, live RTT),
  engine↔moomoo (engine-side gateway probe), engine↔TradeZero (WS ping RTT while
  connected). Current ms + session min/avg/max, threshold-colored.
- **Event log:** timestamped connects/disconnects/reconnect attempts (with backoff
  timing), `StreamGap` reconciles, subscription failures, gate auto-disarms,
  entitlement errors. Logged **engine-side** (SQLite) — survives UI restarts; the
  panel is a live filtered view.

### Chart panel spec

- Lightweight Charts **v5, unforked**. Candles + volume; MACD (and future
  oscillators) in a native v5 sub-pane.
- **Fills-on-chart:** price-anchored diamond markers with border, as a custom v5
  marker plugin — port `drawDiamond`/`hitTestDiamond` (Manhattan hit test, 0.8 size
  factor) from `earlisreal-lightweight-charts` commit `069fa855` and the v3.7.1
  `borderWidth` pattern (`a25e7dc0`). v5's native `atPrice*` positions handle
  culling; buy = soft green, sell = soft pink, thin dark outline (wickplot precedent).
- **Indicators:** per-chart instances from the catalog; UI sends the instance config
  `(symbol, timeframe, type, params)` to the engine, engine streams backfill + live
  series like any other topic. Configs save with the workspace.
- **Live bar handling:** current bar is always in-progress — updated every push,
  finalized on watermark (next-bucket tick). Quiet symbols may hold a partial bar
  past its wall-clock end; the in-progress bar is the last bar updating in place,
  never a "closed" bar. Ingestion is burst-tolerant (first push may deliver many
  bars at once from cache seed/backfill).
- **Cold-symbol state:** sub-minute timeframes show bars growing from subscribe time
  with an explanatory hint; 1m+ backfills immediately (~350 ms full cold open —
  symbol switching must feel instant; brief skeleton at most, no spinners).
- **Sessions:** ET session shading (pre 04:00, RTH 09:30–16:00, post 16:00–20:00);
  intraday aggregation anchored 09:30 ET (anchor configurable engine-side).
- **Interaction (wickplot conventions):** crosshair snaps vertically to bar / floats
  horizontally, cursor-anchored wheel zoom, drag pan with sub-bar accumulation,
  auto-follow right edge with jump-to-live when scrolled back. Dark theme seeded from
  the wickplot palette (`#0F1115` bg, muted green/red up/down).

## Order entry & hotkeys

**Action templates** — one saved recipe, two triggers (hotkey binding and ticket
preset button), edited in one settings screen, stored engine-side:

```
PlaceOrder { side: Buy|Sell|Short|Cover, type: Limit|Market|Stop|StopLimit (default Limit),
             price: Bid|Ask|Last|Mid ± offset, tif: Day|Day+,
             sizing: Dollar($X) | BuyingPowerPct(%) | Shares(N) | PositionFraction(all|half),
             symbol: focus-group symbol }
Management { CancelLast | CancelAll(focused) | CancelAll(everything) | KillSwitch }
```

Sizing resolves at trigger time: `Dollar` → `floor($X / price)`; `BuyingPowerPct` →
`floor(BP × pct / price)` from the live account snapshot; `PositionFraction` → from
the live position (sell-half, flatten).

**Ticket panel:** focused symbol + live bid/ask (click a quote to seed the price
field), side, type (default Limit), price, qty + sizing-mode dropdown, TIF, submit;
preset button row; cancel-all; **kill switch** (distinct, deliberately ugly, always
visible).

**Trigger flow (hotkey = preset = manual):**

1. Resolve template against live quote/account → concrete order + correlation ID.
2. Client-side pre-checks before the wire: qty > 0; stop/stop-limit price coherence
   (TZ does not validate — inverted stop-limits sit unfilled); Market outside RTH
   auto-coerced to Limit-at-last + visible notice (avoids R78).
3. Engine gate envelope (armed, duplicate ID, max order value, position caps,
   open-order cap, day loss). Sync ack `accepted | blocked(reason)`.
4. Orders panel shows `PendingNew` instantly; lifecycle walks via `exec.orders`
   events. Blocked/rejected → prominent toast with the exact reason, never silent.

**Hotkey safety:** order-placing hotkeys fire only while **armed** (disarmed →
"disarmed" flash, nothing sends) and only when an eTape window has OS focus; every
fire flashes the resolved order ("BUY 1,428 AAPL @ 3.50 LMT"). No confirm dialogs —
the gate envelope and arm switch are the protection. **Management hotkeys
(cancel/kill) fire regardless of armed state** — closing exposure is never gated —
and respect TZ rate limits via the engine's token buckets (cancel-all 3/s).

**Cancel/replace:** per-row cancel, cancel-all, re-trigger. Replace renders the
explicit `Replacing` state (cancel confirmed → fresh-ID submit), per the execution
design. Chart-drag amend is out of v1.

## Engine ↔ UI contract additions

The exec topics (`exec.account/positions/orders/fills/status`) and command set are
already specified in the execution design. The UI adds these engine work items:

| Addition | Content |
|---|---|
| `md.quote` / `md.book` / `md.tape` / `md.bars` topics | Per-(symbol, subtype) market data: snapshot then deltas; bars keyed by bucket-start exchange time, partial-bar updates + watermark finalization; engine coalesces to a configurable max rate |
| `md.indicator` topic | Instance subscription `(symbol, timeframe, type, params)` → backfill + live series values |
| `scanner.*` topic | Rank rows + hit events (flash), session-parameterized |
| `news.*` topic | `NewsItem {symbol, headline, source, url, seen_at}` |
| `sys.health` topic | Periodic per-link status + latency (moomoo probe, TZ WS RTT); app-level ping/pong for UI↔engine RTT |
| `sys.events` topic | Append-only event log (connects, gaps, failures, auto-disarms), persisted in SQLite |
| Config store | Workspaces, panel configs, action templates, hotkey bindings, scanner thresholds — CRUD over WS, persisted in SQLite |
| Static file serving | Engine serves the built `ui/dist` |

## Error handling

Honesty policy: **never render stale as live, never render in-flight as done.**

| Edge | Behavior |
|---|---|
| UI↔engine WS drops | Canvases dim + "reconnecting" overlay; on reconnect every topic re-runs snapshot-then-delta; stores rebuild; one clean repaint |
| Engine↔feed degraded | Amber/red dot + tooltip (last event, retry count) from `sys.health`; logged to `sys.events` |
| Quiet symbol | Not staleness — kline pushes only fire on updates; staleness judged from health topics only, never "no messages lately" |
| Order in flight | `PendingNew` until broker confirms (HTTP 200 = delivered, not accepted) |
| TZ reconnect | `StreamGap` badge on orders panel: "state reconciled — verify before acting" |
| Reject / block | Raw R-code text / gate rule name in a prominent toast + orders panel |
| Missing float / no pre-market print | Pending/greyed rendering, never zero |
| Wrong entitlement (delayed data) | Persistent banner, fail loud (premarket-alert lesson) |
| Panel crash | Error boundary → inline error card + reload button; painter crash → unregistered from scheduler, same card; rest of workspace keeps trading |
| Window/tab dies | All state of consequence is engine-side (orders, positions, layouts, tick history); reopen URL → workspace restores in seconds |

## Testing

- **Pure math, unit-tested:** viewport transforms, pan/zoom clamps, nice-number axis,
  bar bucketing/partial-bar mirror of the engine, sizing math, hotkey template
  resolution. Plain TS, no DOM.
- **Golden-image tests:** each painter is `paint(ctx, state)` rendered headlessly
  (node-canvas) against fixture states — full/empty book, partial bar, cold symbol,
  3-digit % — diffed against checked-in PNGs (wickplot's harness pattern).
- **Store/replay tests:** feed stores recorded WS logs (snapshot + deltas + reconnect
  mid-stream), assert final state — the UI twin of the engine's `replay(log) == state`
  invariant. Fixtures captured from real engine output so contract drift fails tests.
- **Ticket safety tests:** table-driven — every sizing mode × price source,
  disarmed-blocks-hotkey, RTH coercion, inverted stop-limit rejection. Densest
  coverage; this code touches money.
- **Smoke E2E (Playwright, few and boring):** engine in sim/replay mode → both
  workspaces → link-group focus → charts populate, ladder paints, paper order walks
  `PendingNew → New → Filled`.

## Out of scope (v1)

Ladder click-trading; chart-drag order amend; drawing tools; chart trade-replay;
halt feed (visual slot reserved); news enrichment (epoch timestamps); short-locate UI
flow (needs live TZ verification first — the 30 s accept countdown design exists in
the TZ doc for when it lands); workspace manager UI (create/rename/clone/delete);
per-workspace theming; import/export; sounds/audio alerts; options; multi-account.
